# Evidence Map Template

Use this table for every `Research Meeting Pack`.

| Claim / 结论 | Supporting Evidence / 支持证据 | Counter-evidence / 反例或冲突 | Strength | Manual Confirmation | Next Action |
|---|---|---|---|---|---|
|  |  |  | Strong / Moderate / Weak / Insufficient |  |  |

## How to Fill

- `Claim / 结论`: Write a meeting-ready claim, not a vague topic.
- `Supporting Evidence / 支持证据`: Cite the available evidence type, such as paper, metrics, figures, notes, progress, or git.
- `Counter-evidence / 反例或冲突`: Include anomalies, metric conflicts, failure cases, missing repeats, missing baselines, or assumptions that could weaken the claim.
- `Strength`: Use `Strong`, `Moderate`, `Weak`, or `Insufficient`.
- `Manual Confirmation`: State what should be manually checked.
- `Next Action`: State the next concrete verification step.

If no direct counter-evidence appears in the current materials, write:

> Current materials provide no direct counter-evidence, but this still needs confirmation by ...

Do not leave counter-evidence blank.
