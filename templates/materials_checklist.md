# Materials Checklist

Use this checklist before asking an AI assistant to prepare a research meeting pack.

## papers/

Put PDF papers or pasted paper excerpts here.

Use for:

- research background
- method keywords
- related work
- assumptions to verify
- paper cards

## experiment_results/

Put CSV, JSON, tables, metric logs, or copied result snippets here.

Use for:

- before/after comparison
- metric trends
- scene-level failure analysis
- evidence for experiment claims
- next experiment planning

## figures/

Put result images, plots, before-after comparisons, screenshots, and failure cases here.

Use for:

- visual evidence
- slide figures
- qualitative comparison
- failure case discussion
- explaining metrics more concretely

## notes.md

Write rough observations, meeting notes, advisor feedback, or experiment comments.

Use for:

- context that is not visible in tables
- interpretation hints
- known issues
- discussion questions

## progress.md

Write explicit weekly progress if available.

Use for:

- what changed this week
- implementation progress
- experiment status
- next actions

If this file is empty, the AI should infer the reportable thread from other materials.

## git commits

Paste recent commit messages or ask Codex/Cursor to inspect git history.

Use for:

- implementation evidence
- experiment script changes
- data processing updates
- reproducibility notes
